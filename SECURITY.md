# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability, please report it responsibly:

- **Preferred:** Open a [GitHub Security Advisory](https://github.com/darshangoswami/syncify/security/advisories/new)
- **Alternative:** Email security concerns to the repository owner directly

**Do not** open a public issue for security vulnerabilities.

## Scope

The following areas are in scope for security reports:

- OAuth authentication flow (Spotify, TIDAL)
- Cookie signing and session management
- Invite approval and access gating
- API route authorization
- Cross-site request forgery (CSRF) protections
- Input validation and injection vulnerabilities

## Response timeline

- **Acknowledgement:** within 48 hours
- **Initial assessment:** within 1 week
- **Fix or mitigation:** best effort, depending on severity

## Supported versions

Only the latest version on the `main` branch is supported.
